Quick Feedback

The simplest arrangement a man can make. Validate the input. Save it to disk. No emails, no webhooks, no moving parts. Just the bones.

This form demands a PHP server to function. Run php -S localhost:8000 in the project folder, then open http://localhost:8000/demo1.html — lest you stare at an empty page and blame the wrong party.

What works & what needs setup

✓ Form submission and file storage work out of the box. No configuration needed.
📁 Submissions are saved as JSON files in submissions/demo-file/.

What happens when you submit

  • ✓ Server validates every field — required, minlength, maxlength. The client's opinion is noted and ignored.
  • 💾 Submission stored as a JSON file. Quiet, orderly, no complications.

That is the full extent of it. No emails dispatched, no webhooks fired, no third parties notified. The simplest configuration this engine offers — and often the wisest place to start.

Where does the data go?

submissions/demo-file/{timestamp}_{id}.json

A JSON file for each submission. Tidy. Readable. Under your control.

The on_submit configuration

"on_submit": { "store": true }

One line. One truth. A man who needs more can look at Demo 2 and Demo 3.

ⓘ Nothing leaves the server. No data is sent anywhere. Files are saved to the submissions/ folder and that is where they stay — until you decide otherwise.
