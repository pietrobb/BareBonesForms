Advanced Features

A multi-page job application form. Sections, conditional logic, star ratings, column layout, email confirmation, "other" option — the full arsenal. Three pages, one submission.

This form demands a PHP server to function. Run php -S localhost:8000 in the project folder, then open http://localhost:8000/demo4.html — lest you stare at an empty page and blame the wrong party.

What works & what needs setup

✓ Form submission, storage, conditional logic, and multi-page all work out of the box.
📁 Submissions are saved as JSON files in submissions/demo-advanced/.
⚙ Confirmation email to the applicant — to enable, open config.php and set: method → "smtp", smtp_host, smtp_port, smtp_user, smtp_pass (your mail server credentials), from_email (sender address). The confirmation is sent to the email entered in the form. Template and subject are defined in forms/demo-advanced.json under on_submit.confirm.

What happens when you submit

  • 1️⃣ Page 1 validates personal info — name, email confirmation match, phone, URL.
  • 2️⃣ Page 2 validates position details — required fields, date range, conditional office field.
  • 3️⃣ Page 3 validates skills — ratings, cover letter (50+ chars), pattern-matched referral code, consent.
  • ✓ Server validates everything again, then stores the submission.

Hidden fields (like "Preferred Office" when Remote is selected) are excluded from both validation and the submitted data. They simply cease to exist.

Conditional logic in action

On page 2, choose "Remote" and the "Preferred Office" dropdown disappears. Choose "Hybrid" or "On-site" and it reappears. The hidden field is removed from the submission entirely.

"show_if": { "field": "work_type", "value": "remote", "op": "not" }

Translation: show this field when work_type is NOT "remote".

Multi-page forms

Insert a page_break field to split the form. Each page validates independently before you can advance. The submit button appears only on the last page.

{ "name": "page2", "type": "page_break" }

Previous/Next button labels are translatable via language packs.
