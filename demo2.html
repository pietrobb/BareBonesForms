Event Registration

Store submissions and notify the administrator by email. A step beyond mere storage — now someone is informed that a newcomer has arrived at the camp.

This form demands a PHP server. Run php -S localhost:8000 in the project folder, then open http://localhost:8000/demo2.html

What works & what needs setup

✓ Form submission and storage work out of the box.
📁 Submissions are appended to submissions/demo-csv/demo-csv.csv (CSV format).
⚙ Admin email notification — open config.php and set: method → "smtp", smtp_host, smtp_port, smtp_user, smtp_pass (your mail server credentials), from_email (sender address). The recipient is in forms/demo-csv.json → on_submit.notify.to.

What happens when you submit

  • ✓ Server validates all fields — including multi-checkbox groups, which are submitted as arrays and verified against allowed options.
  • 💾 Submission stored (file, SQLite, MySQL, or CSV — whichever you've set in config.php).
  • ✉ Admin notification dispatched to the address specified in the form definition.

Email details

To: admin@example.com
Subject: New registration: {{name}} ({{ticket}})
Template: templates/notify.html

The subject line uses {{placeholders}} — filled with actual field values at the moment of sending. So you know exactly who walked through the door and what ticket they hold.

The on_submit configuration

"on_submit": { "store": true, "notify": { "to": "admin@example.com", "subject": "New registration: {{name}}", "template": "notify.html" } }
ⓘ admin@example.com is an IANA-reserved domain — mail sent there is silently dropped by design. On localhost, PHP mail() typically has no mail transfer agent configured, so nothing leaves the machine. To test with real emails, set up SMTP in config.php and replace the address with your own.
