Bug Report

The full pipeline. Validate, store, email the admin, fire a webhook, send a confirmation to the reporter. Everything this engine can do — in one form. A demonstration of the complete machinery.

This form demands a PHP server. Run php -S localhost:8000 in the project folder, then open http://localhost:8000/demo3.html

What works & what needs setup

✓ Form submission and storage work out of the box.
📁 Submissions are saved as JSON files in submissions/demo-webhook/.
⚙ Admin email notification — to enable, open config.php and set: method → "smtp", smtp_host, smtp_port, smtp_user, smtp_pass (your mail server credentials), from_email (sender address). The recipient address is set in forms/demo-webhook.json under on_submit.notify.to.
⚙ Webhook delivery — open config.php and set webhook_secret (any random string for HMAC signing). The target URL is set in forms/demo-webhook.json under on_submit.webhook.url.

What happens when you submit

  • ✓ Server validates all fields — including minlength on textareas, because a bug report that says "it broke" helps no one.
  • 💾 Submission stored to whichever backend you've configured.
  • ✉ Admin notification — "Bug Report [critical]: from Jane" — so you know the severity before you open it.
  • 🌐 Webhook POST fired to the configured URL with an HMAC-SHA256 signature, so the receiving end knows the message is genuine.
  • ✉ Confirmation email to the reporter, acknowledging receipt. Common courtesy — even in Deadwood.

Where the webhook goes

URL: httpbin.org/post
Method: POST with JSON body
Signature: HMAC-SHA256 in X-BBF-Signature header

Where the emails go

Admin: admin@example.com — "Bug Report [{{severity}}]: from {{reporter}}"
Confirmation: {{email}} — whatever address the reporter provides

The on_submit configuration

"on_submit": { "store": true, "notify": { "to": "admin@example.com", "subject": "Bug Report [{{severity}}]: ...", "template": "notify.html" }, "webhooks": ["https://httpbin.org/post"], "confirm_email": { "to": "{{email}}", "subject": "Bug report received", "template": "confirm.html" } }
ⓘ httpbin.org is a public HTTP testing service — it echoes the request back and stores nothing. It is the equivalent of shouting into a canyon; you hear it come back, but it goes nowhere. Emails to example.com are silently dropped (IANA-reserved domain). On localhost without SMTP configured, no email leaves the machine. The confirmation email would go to whatever address you type above — but only if SMTP is properly set up in config.php.
