Allergy Remedy Finder

A complex multi-page form with conditional logic, field groups, nested conditions, and client-side scoring. Select your symptoms — only relevant follow-up questions appear. At the end, a ranked remedy profile is calculated from your answers.

This form demands a PHP server to function. Run php -S localhost:8000 in the project folder, then open http://localhost:8000/demo5.html — lest you stare at an empty page and blame the wrong party.

What works & what needs setup

✓ Fully functional out of the box. No configuration needed.
📁 Submissions are saved as JSON files in submissions/demo-allergy/.

How it works

  • 1️⃣ Page 1: Select your main allergy symptoms.
  • 2️⃣ Page 2: Detail each symptom — only selected symptoms get follow-up questions (conditional groups).
  • 3️⃣ Page 3: General modalities — what makes things better or worse, mood, thirst.
  • ✓ Each answer adds weighted points to matching remedies. Top matches are ranked and displayed.

Features demonstrated

  • multi-page (3 pages)
  • field groups with show_if

  • checkbox-driven visibility
  • conditional symptom tunnels
  • inline & column layouts
  • cross-page condition evaluation
  • client-side scoring engine
  • warning group (css_class)

Conditional logic in action

Page 2 uses field groups — each group appears only when its symptom is selected on page 1:

"type": "group", "show_if": { "field": "symptoms", "value": "sneezing" }

Select only "runny nose" and "itchy eyes" on page 1, and page 2 shows only those two groups. The form leads you down relevant paths only.

Nested conditions

The warning on page 3 uses any + all nesting:

"show_if": { "any": [ { "field": "symptoms", "value": "skin_rash" }, { "all": [ { "field": "symptoms", "value": "sneezing" }, { "field": "sneeze_intensity", "value": "frequent_violent" } ]} ] }

Translation: show warning if skin rash is selected, or if sneezing is selected and intensity is violent.
